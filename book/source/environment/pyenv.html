pyenv管理python版本

场景

在实际学习和开发的场景中，我们常常会遇到这么一些情况:

  • 系统自带了python2.x的版本，但是我们需要python3.x的版本
  • 系统目前已经安装了一些python版本，但是我们想要体验一下新版本，或者一些其它发行版的python，例如科学计算常用的Anaconda。

这时候我们就会想到，应该怎么去下载、安装和管理这么多的python呢？如果当我们想要卸载某个版本的时候，又应该怎么去操作呢？基本的原则就是: 要优雅。这也是我为什么喜欢用macOS或者linux来开发的原因。macOS下有homebrew，linux下的各个发行版也有对应的包管理工具。在安装和卸载东西的时候，会自动安装或者卸载相应的依赖。相比起Windows需要各种手动安装和配置环境变量，卸载后又不知道卸载干净没有，包管理器简直就是强迫症的福音。

介绍

pyenv是一个python版本的管理器，通过它我们可以:

  • 更改当前用户全局的python版本
  • 为每一个python项目，分配一个唯一的python版本。且不同目录间不会相互影响，所安装的包也不会相互影响。

更详细的可以参考pyenv的Github官网

安装

$ git clone https://github.com/pyenv/pyenv.git ~/.pyenv

对于mac,可以使用homebrew来安装:

brew install pyenv

配置

对于bash

$ echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
$ echo 'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
$ echo 'eval "$(pyenv init -)"' >> ~/.bashrc

对于zsh

$ echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.zshrc
$ echo 'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.zshrc
$ echo 'eval "$(pyenv init -)"' >> ~/.zshrc

然后重新加载shell

$ exec $SHELL -l

使用

可以在shell里输入pyenv --help或者在pyenv commands查看:

查看正在使用的python版本

$ pyenv version

查看系统中可供pyenv使用的python版本

$ pyenv versions

列出所有可以安装的python版本

$ pyenv install --list

安装某个版本的python

$ pyenv install <python版本>

安装的版本会在~/.pyenv/versions目录下。
如果下载速度过慢，或者文件较大，可以先到官网或者一些国内的镜像网站下载，然后将文件放在放在~/.pyenv/cache目录下，再执行安装。

卸载某个版本的python

$ pyenv uninstall <python版本>

设置全局python版本

$ pyenv global <python版本>

设置局部python版本

$ pyenv local <python版本>

查看是否生效which python或者python --version

卸载

  1. 如果要暂停使用pyenv,先把pyenv init从shell的启动文件里删除。例如你安装的时候是添加到.bashrc里，就到.bashrc里把这一句删除。
  2. 如果要完全删除pyenv，在第一步的基础上，执行
    rm -rf $(pyenv root)
    如果你是用包管理工具安装的pyenv，就可以使用包管理工具来卸载，对于Homebrew:
    brew uninstall pyenv
